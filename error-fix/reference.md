# Error-fix - Reference
本文件为技能 `error-fix` 提供最小必要的错误分类、排查方法与执行闭环。

---

## 一、错误类型（最小必要分类）

1. 语法错误  
   - 代码不符合语法规则（缺括号、拼写、缩进）

2. 运行时错误  
   - 执行中崩溃（空引用、除零、越界、内存溢出）

3. 逻辑错误  
   - 运行正常但结果错误（算法/条件/循环错误）

4. 网络错误  
   - 请求/响应异常（超时、DNS 失败、连接拒绝）

5. 系统错误  
   - 环境或系统问题（权限、路径、文件、资源不足）

---

## 二、排查方法（最小必要工具集）

> 使用方式：**先分类 → 再排查 → 再验证**

1. 日志分析  
   - 查看应用/系统日志定位报错点。

2. 分步调试  
   - 将流程拆解为小步骤逐一执行。

3. 环境与配置检查  
   - 核对依赖、版本、路径、权限、网络、环境差异。

4. 对比分析  
   - 与“已知正常”的版本/环境对比差异。

5. 根因分析（5W1H）  
   - 从 What/When/Where/Who/Why/How 收敛到单一根因。

6. 测试验证  
   - 修复后通过用例或回归测试确认问题解决。

---

## 三、最小排查流程（逻辑闭环）

1. **确定错误类型**（语法 / 运行时 / 逻辑 / 网络 / 系统）  
2. **按类型选择排查方法组合**  
   - 语法 → 日志 + 分步调试  
   - 运行时 → 日志 + 环境检查  
   - 逻辑 → 对比分析 + 测试验证  
   - 网络 → 配置检查 + 环境检查  
   - 系统 → 权限/路径检查  
3. **修复后验证**（测试验证）
