# sry-skills

个人收集的技能（Skill）仓库，包含多种工程自动化技能，可用于代码简化、错误修复、前端设计、UI/UX优化等多种场景。

## 项目介绍

sry-skills是一个基于个人收集的技能（Skill）仓库，包含了多种用于工程自动化的技能定义和实现。每个技能专注于特定领域的任务，可独立使用或组合使用，帮助用户快速完成各种工程任务。

## 核心特点

- **技能丰富**：包含9种不同领域的技能，涵盖代码、设计、测试等多个方面
- **独立封装**：每个技能独立封装，可单独使用
- **易于扩展**：采用模块化设计，便于添加新技能
- **详细文档**：每个技能都有完整的使用说明和示例
- **多种应用场景**：适用于开发、设计、测试、调试等多种工程场景

## 项目结构

```
sry-skills/

```

## 技能列表

### code-simplifier
- **名称**：代码简化器
- **描述**：简化和优化代码，提高代码的清晰度、一致性和可维护性，同时保持所有功能不变
- **使用场景**：
  - 简化复杂代码结构
  - 优化代码可读性
  - 统一代码风格
  - 移除冗余代码

### error-fix
- **名称**：错误修复专家
- **描述**：根据用户提供的错误信息，执行完整的错误排查与修复流程，包括错误识别、根因分析、修复方案制定和执行
- **使用场景**：
  - 开发人员排查代码错误
  - 系统管理员分析系统故障
  - 技术支持人员快速定位问题
  - 普通用户解决软件问题

### failure-analysis-taskforce
- **名称**：故障分析任务组
- **描述**：诊断AI IDE执行失败、无效果或结果不理想的原因，并提供可执行的修复方案与提示词优化策略
- **使用场景**：
  - AI IDE执行失败时的故障分析
  - 优化AI IDE执行效果
  - 改进提示词设计
  - 提升AI IDE执行成功率

### frontend-design
- **名称**：前端设计专家
- **描述**：创建独特、高质量的前端界面，避免通用AI美学，注重设计细节和创意选择
- **使用场景**：
  - 构建Web组件
  - 设计网页或应用界面
  - 创建HTML/CSS布局
  - 美化Web UI

### json-canvas
- **名称**：JSON Canvas编辑器
- **描述**：创建和编辑包含节点、连线、分组及关联关系的JSON画布文件（.canvas格式）
- **使用场景**：
  - 制作思维导图
  - 创建流程图
  - 设计可视化画布
  - 在Obsidian中使用画布文件

### summary-specialist-taskforce
- **名称**：总结专家任务组
- **描述**：从对话中提炼成功经验与失败教训，自动去重、分组、合并，并持续写入统一文档
- **使用场景**：
  - 总结AI IDE执行经验
  - 提炼成功模式与失败模式
  - 维护知识库
  - 更新统一经验文档

### ui-ux-pro-max
- **名称**：UI/UX设计智能助手
- **描述**：包含UI风格、调色板、字体搭配、图表类型、产品推荐、UX指南和特定技术栈最佳实践的可搜索数据库
- **使用场景**：
  - 设计UI界面
  - 选择配色方案
  - 搭配字体
  - 设计响应式布局
  - 优化用户体验

### web-artifacts-builder
- **名称**：Web制品构建器
- **描述**：使用现代前端技术构建复杂的Web制品，支持React、TypeScript、Tailwind CSS和shadcn/ui
- **使用场景**：
  - 构建复杂的Web界面
  - 创建React应用
  - 打包单文件HTML制品
  - 开发交互式Web组件

### webapp-testing
- **名称**：Web应用测试专家
- **描述**：提供Web应用测试的示例和脚本，支持控制台日志记录、元素发现和静态HTML自动化
- **使用场景**：
  - 测试Web应用功能
  - 自动化浏览器操作
  - 验证页面元素
  - 记录测试结果

## 技能定义格式

每个技能都包含以下核心部分：

1. **名称（name）**：技能的唯一标识符
2. **描述（description）**：技能的功能和用途
3. **使用场景**：技能适用的具体场景
4. **输入（inputs）**：技能所需的输入参数
5. **输出（outputs）**：技能执行后的输出结果
6. **执行步骤（steps）**：技能的具体执行流程
7. **前置条件（preconditions）**：执行技能前必须满足的条件

## 使用方法

1. 选择需要的技能
2. 查看该技能的SKILL.md文件，了解详细使用方法
3. 根据技能要求提供输入
4. 执行技能并获取输出
